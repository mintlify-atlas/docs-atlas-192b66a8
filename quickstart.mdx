---
title: "Quickstart"
description: "Map your first objects with Mapster in five steps: define POCOs, call Adapt, map in place, and add custom configuration."
---

This guide walks you through the core Mapster workflow from scratch. By the end, you will know how to map objects, mutate existing instances, and apply per-type configuration rules.

<Steps>

  <Step title="Define your source and destination POCOs">
    Create two plain C# classes that represent your source and destination types. Mapster automatically maps properties that share a name and a compatible type.

    ```csharp
    public class Student
    {
        public int    Id        { get; set; }
        public string FirstName { get; set; }
        public string LastName  { get; set; }
        public int    Age       { get; set; }
    }

    public class StudentDto
    {
        public int    Id       { get; set; }
        public string FullName { get; set; }
        public int    Age      { get; set; }
    }
    ```

    <Note>
      You do not need to annotate your classes or register anything for basic mapping. Mapster discovers matching properties by convention.
    </Note>
  </Step>

  <Step title="Map to a new object">
    Add `using Mapster;` to your file, then call `Adapt<TDestination>()` on any source object. Mapster creates and populates the destination instance for you.

    ```csharp
    using Mapster;

    var student = new Student
    {
        Id        = 1,
        FirstName = "John",
        LastName  = "Doe",
        Age       = 21
    };

    var dto = student.Adapt<StudentDto>();
    // dto.Id  == 1
    // dto.Age == 21
    // dto.FullName is null — no config yet for the combined name
    ```

    Properties that exist on the source but not the destination (and vice versa) are silently skipped.
  </Step>

  <Step title="Map to an existing object">
    When you already hold a destination instance — for example, one fetched from a cache or partially populated elsewhere — pass it directly to `Adapt`. Mapster updates it in place.

    ```csharp
    var existingDto = new StudentDto { Id = 99 };

    student.Adapt(existingDto);
    // existingDto.Id  is now 1  (overwritten from source)
    // existingDto.Age is now 21
    ```

    <Tip>
      This overload is useful when you want to merge data from a source object into an existing destination without allocating a new instance.
    </Tip>
  </Step>

  <Step title="Add per-type configuration">
    Use `TypeAdapterConfig<TSource, TDestination>.NewConfig()` to customise mapping for a specific type pair. Call this once at application startup — for example, in `Program.cs` or a dedicated mapping bootstrap method.

    The example below ignores the `Age` property and combines `FirstName` and `LastName` into `FullName`:

    ```csharp
    TypeAdapterConfig<Student, StudentDto>
        .NewConfig()
        .Ignore(dest => dest.Age)
        .Map(dest => dest.FullName,
             src => string.Format("{0} {1}", src.FirstName, src.LastName));
    ```

    Now adapt again with the same call:

    ```csharp
    var dto = student.Adapt<StudentDto>();
    // dto.Id       == 1
    // dto.FullName == "John Doe"
    // dto.Age      == 0  (ignored — left at default)
    ```

    <Note>
      Calling `NewConfig` replaces any existing configuration for this type pair. If you want to extend an existing mapping instead of replacing it, use `ForType` instead.
    </Note>
  </Step>
