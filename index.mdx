Mapster is a high-performance object mapper for .NET. It maps between C# types by convention — no boilerplate required. When you need more control, it provides a rich fluent API for custom mappings, constructor mapping, value transforms, and more. For maximum performance, use code generation to produce static mapper classes at build time.

## Why Mapster?

Mapster is designed to be fast, memory-efficient, and easy to use. Benchmark results comparing 10,000 object mappings:

| Library | Mean | Allocated |
|---|---|---|
| Mapster 6 (codegen) | 34 ms | 124 MB |
| Mapster 6 (FEC) | 37 ms | 118 MB |
| Mapster 6 | 109 ms | 124 MB |
| ExpressMapper 1.9.1 | 206 ms | 237 MB |
| AutoMapper 10.0.0 | 421 ms | 351 MB |

## Basic usage

Map an object to a new instance with a single method call:

```csharp
var destObject = sourceObject.Adapt<Destination>();
```

Map into an existing object:

```csharp
sourceObject.Adapt(destObject);
```

Project a queryable to a DTO type without materializing the full entity:

```csharp
var dtos = dbContext.Users.ProjectToType<UserDto>().ToList();
```

<Steps>
  <Step title="Install Mapster">
    ```bash
    dotnet add package Mapster
    ```
  </Step>
  <Step title="Map your first object">
    Call `.Adapt<TDestination>()` on any source object. Mapster maps matching property names by convention — no configuration needed.
  </Step>
  <Step title="Add custom configuration (optional)">
    Use `TypeAdapterConfig<TSource, TDestination>.NewConfig()` to define per-type rules like ignored fields, computed values, or custom resolvers.
  </Step>
  <Step title="Generate code for production (optional)">
    Install `Mapster.Tool` and add a build target to generate static mapper classes for compile-time safety and maximum performance.
  </Step>
</Steps>
